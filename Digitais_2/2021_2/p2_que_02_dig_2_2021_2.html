Abaixo segue um exemplo de solução esperada:

Note que um FF é acrescentado ao circuito para habilitar ou desabilitar sinal de clock para o registrador de deslocamento (sinal ENABLE).

Note que enquanto a saída deste FF permanece resetado (Q=0; ENABLE=0), o sinal de Clock não ingeressa no registrador de deslocamento e ao mesmo tempo, o sinal de ENABLE=0 é usado para gerar (permanentemente) um sinal de "Master Reset" para o Registrador de Deslocamento, o que garante que todas as suas saída permanecem desativadas logo que o circuito é energizado.

Note que o sinal de "Reset" para o Flip-Flop é gerado de 2 formas:

  • quando o circuito é recém energizado (sinal PWR_UP―=0) ou;
  • quando a última saída do Registrador de Deslocamento é ativado (Q7=1, ou Q7―=0).

Usando equações:

Reset|74LS74=Q7+PWR_UP (raciocínio inicial com todos os sinais "Ativo Alto").

Adaptando ao nosso circuito, necessitamos que este sinal de Reset seja "ativo baixo", já temos PWR_UP― e só nos faltaria Q7― (fácil de obter via porta inversora). Equacionando:

Reset|74LS74=Q7+PWR_UP―

Usando DeMorgan: x+y―=x―⋅y―, então:

Reset|74LS74=Q7―⋅PWR_UP―.

Note que quando o usuário pressiona o botão de START, está sendo comandado um "Set" no Flip-Flop (de controle de circuito), o que habilita o sinal ENABLE (ENABLE=1). Isto desativa o Master Reset e libera o ingresso do sinal de Clock ao Registrador de Deslocamento e assim têm início o efeito visual (leds avançando até último estágio).

Note que depois que o último led é ativado (D7 na figura anterior), a saída Q7 é ativada (Q7=1). E este sinal é aproveitando então para resetar o Flip-Flop que controla o circuito, fazendo o mesmo "voltar" para a condição inicial que é só comandar o efeito quando o usuário pressionar o botão de Start.

Desta forma o efeito visual apenas inicia quando o usuário pressiona o botão de Start e "para automaticamente" depois que o último led é ativado.

Observação final: esta solução é regatilhável, isto é, se usuário pressionar o botão de Start durante a execução do efeito visual, o mesmo recomeça.

A solução para este problema é simples porque a modificação exigida implica apenas controlar a operação do circuito entre 2 estados: ocioso/parado x executando efeito visual. Então um simples Flip-Flop pode ser usado para controlar a alternância do circuito entre estes 2 estados. Se por acaso, outro circuito sequencial implicasse mais estados de operação, mais FFs teriam de ser usados e suas saídas eventualmente decodificadas para gerar linhas de controle internas (sinais de "Enable" interno) para habilitar/desabilitar partes de um circuito mais complexo conforme o estado assumido pelo mesmo.
